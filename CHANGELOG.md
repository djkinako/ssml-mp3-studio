# 更新履歴

このファイルは SSML MP3 Studio の更新履歴。
バージョン番号は [Semantic Versioning](https://semver.org/lang/ja/) に準拠（`MAJOR.MINOR.PATCH`）:
- **PATCH** (0.0.x): バグ修正・微調整
- **MINOR** (0.x.0): 新機能・UI 変更
- **MAJOR** (x.0.0): 互換性のない破壊的変更

## [3.2.0] - 2026-06-16

### 追加(双方向ベース言語対応)
- 🌐 **buildSsml() を双方向対応化**: 右上の UI 言語切替(🇯🇵JA / 🇹🇼繁)で SSML の **「地の文の言語」を切り替え可能**に
  - 🇯🇵 **JA モード**(日本人向け): 地の文 = 日本語、`<lang xml:lang="zh|cmn-*">` を中国語声に置換(従来通り)
  - 🇹🇼 **繁モード**(台湾人向け): 地の文 = 中国語(繁体字)、`<lang xml:lang="ja-*">` を日本語声に置換 ✨ 新規
- 🎯 **サンプル SSML も言語連動**: 🇯🇵JA 時は日本語ベース、🇹🇼繁 時は中国語繁体字ベースのサンプルを投入
- 🛠️ Functions のリクエスト body に `baseLang: "ja"|"zh"` を追加(未指定なら "ja" で後方互換)
- 🧮 `estimateBuiltSize()` も baseLang 対応(送信時バイト計算が両モードで正確に)

### 削除
- 🧹 不要 UI: `(サンプル: 台湾華語 N1-N2 学習向け教材・約 1 分)` キャプション削除(JA/TW 両 i18n から)

### 内部
- 🏷️ VERSION 3.1.5 → 3.2.0(MINOR、機能追加)
- 🔄 キャッシュバスター `?v=3.2.0` に更新

### 背景(なぜ MINOR バンプか)
台湾人向けの台湾華語ベース SSML(地の文が繁体字、`<lang xml:lang="ja-JP">` で日本語埋め込み)が今までは「地の文=日本語」決め打ちでうまく読まれなかった。`dual-v5.md` プロンプトの「Q1 母語」が既にこの双方向対応を前提に書かれていたが、ツール側が未対応だった部分を補完。これでプロンプトの設計通りに動くようになる。

## [3.1.2] - 2026-06-16

### 修正
- 🐛 **「ユーザー表示は4608バイトなのに Google TTS が5000超え」と弾くバグ修正**
  - 原因: `buildSsml()` で `<lang>` → `<voice><prosody>` 変換時にサイズ膨張(`<lang>` 1個 ≒ +40バイト)。きなこ事例では `<lang>` 27 個で +1110バイト膨らんで実送信サイズ 5719 バイトに
  - 対策 A(両ホスト): **速度 100% (1.0) のときは `<prosody>` を省略**して 1 個あたり -21 バイト削減
    - `<voice name="X"><prosody rate="100%">XXX</prosody></voice>`(71+X) → `<voice name="X">XXX</voice>`(50+X)
    - 外側の `<speak><prosody rate="100%">…</prosody></speak>` も同様省略
    - これで 27 個の `<lang>` 含む SSML が 5719 → 5152 バイトに削減(まだ超えるが、改善は大きい)
  - 対策 B(フロント): **`updateByteCount()` を「元サイズ + 送信時サイズ」両方表示**に改修
    - 「4608 バイト(送信時 5719, 上限 5000)」の形式
    - 赤字判定は**送信時サイズ**ベースに変更(これが実際の Google TTS 制限)
    - 声・速度変更時にも再計算するリスナー追加
  - 🏷️ VERSION 3.1.1 → 3.1.2(PATCH、UX 改善 + バイト計算修正)

### Long Audio API について(きなこ質問)
- 料金は通常 API と同じ(Standard $4 / 100万文字、WaveNet $16 / 100万文字)
- 文字数上限が 5000 → **100万バイト**まで拡張、ただし**非同期**で GCS バケットに書き出し
- 実装複雑度高、学習教材用途にはオーバースペック → 当面採用せず

### 設計判断の背景(Issue #17 ハカセ&ケンタ会話劇から)

きなこの本質的観察: **「字形構成解説なんて、ながら聞きで再構成できない」**

> 「『帰国の帰』『材料の材』だけで十分機能する」
> 「『ニンベンに門』って言われても紙ないと書けへんし、頭にも浮かばへんわ」

→ **音声学習で字形構成解説は そもそも機能していない**(目で見ないと再構成できない)。書取練習(紙とペン)で初めて意味を持つ。**ながら聞きで「言わない方が学習効率高い」**。

→ Phase 2 から構成解説を全削除することで:
1. 教材品質は維持(きなこ観察通り「帰国の帰」で十分)
2. エージェント裁量が極小化 → 副作用リスク減
3. ルール数が減る → 新ルール隙間狙いが構造的に発生しない(v1.4.1 と相補)

## [0.8.1] - 2026-06-14

### 修正
- 🐛 **効果音再生で MP3 停止するバグを修正**。原因は MP3 フォーマット不一致(効果音: 44.1kHz JntStereo 128/192kbps + ID3v2.4 タグ / Google TTS: 24kHz mono 64kbps + ID3 なし)で、連結時にデコーダがフレームヘッダー切替で詰まっていた

- 🔧 `app.js`: TTS API リクエストの `audioConfig` に `sampleRateHertz: 24000` を明示。Google TTS の出力フォーマットを今後も 24kHz mono に固定して効果音と完全一致を保証

### 背景
きなこの実機検証で「効果音を再生する部分で MP3 が止まる」報告を受け、フォーマット不整合を特定。`file` コマンドで MP3 ヘッダー解析、`xxd` で ID3 タグの有無確認、ffmpeg で 24kHz mono への再エンコードという順で修正。
