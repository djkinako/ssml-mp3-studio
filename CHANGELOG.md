# 更新履歴

このファイルは SSML MP3 Studio の更新履歴。
バージョン番号は [Semantic Versioning](https://semver.org/lang/ja/) に準拠（`MAJOR.MINOR.PATCH`）:
- **PATCH** (0.0.x): バグ修正・微調整
- **MINOR** (0.x.0): 新機能・UI 変更
- **MAJOR** (x.0.0): 互換性のない破壊的変更

---

## [0.6.0] - 2026-06-14

### 追加
- 🎯 **複数 SSML ブロック対応**: UI の SSML 入力エリアを「ブロック」単位に変更。**「+ ブロックを追加」ボタンで動的に textarea を増やせる**(削除も可、最低 1 ブロック)。各ブロックに独立した SSML を貼り付ける
- 🔗 **複数 MP3 自動結合**: 「🔊 MP3 生成」押下時、全ブロックを順次 Google TTS API に投げて MP3 を取得 → **`Uint8Array` で単純連結して 1 つの MP3 として DL**。ffmpeg / Web Audio API 不要、MP3 フレームの独立性を利用
- 📊 各ブロックの個別バイトカウンタ + 全ブロック合計バイトカウンタ表示
- 📈 進捗表示: 「ブロック X を生成中… (i/N)」のように順次状況を表示

### 変更
- 📋 `prompts/zhtw-v4.md` v4.2: v0.5.0 で入れた「全ペア統合」ルールを廃止し、**v0.4.0 ルール(ペアごとに独立した `<speak>`)に戻す**。理由: アプリ側で MP3 結合機能を持ったため、SSML レベルでの統合は不要。各ペアを 5000 バイト上限内のまま最高品質で出せる
- 📋 `CLAUDE.md` の Issue 対応コメントフォーマット例も「ペアごと別 `<speak>` + 各ブロックに貼る運用」に更新

### 背景
v0.5.0 では「複数ペアを 1 つの `<speak>` に統合」というプロンプトレイヤーの対応を入れたが、Issue #1 の 3 ペア依頼で **物理的に統合できない**(合計 8358 バイトで 5000 上限の 1.7 倍)ことが判明。アプリ側で「複数 SSML → 1 MP3」を解決する方が本質的かつスケーラブルと判断し、v0.6.0 で実装。プロンプト v4 のシンプルさも保てる。

## [0.5.0] - 2026-06-14

### 変更
- 🎯 **複数ペアの SSML を 1 つの `<speak>` に統合**するルールに変更(ssml-mp3-studio が 1 SSML = 1 MP3 として処理する前提に合わせる)。ペア間に「次いくで。」のような区切りアナウンス + `<break time="4s"/>` を挟む
- 📋 `prompts/zhtw-v4.md` v4.1: 「複数ペア統合の構造テンプレート」「SSML サイズ制限への対応(5000バイト超は自動分割+明示)」「セルフレビュー6項目(`<speak>` 統一・サイズ上限チェックを追加)」を新設
- 📋 `CLAUDE.md` の Issue 対応コメントフォーマット例を統合版に更新、5000バイト超時の分割表示ルールを追記

### 背景
v0.4.0 リリース後の初回ユーザーテスト(Issue #1)で、3 ペア依頼に対して各ペアを別 `<speak>` で返したところ「複数ペアは 1 MP3 にまとめたい」とフィードバック。ssml-mp3-studio は SSML 入力欄が 1 つで 1 `<speak>` 前提のため、ペアを統合する方が UX として正しい。

## [0.4.0] - 2026-06-14

### 追加
- 🎧 **Issue 起点 SSML 生成ワークフロー** を導入。日本語+台湾華語(繁体字)ペアを GitHub Issue で投げると、Claude Code が `prompts/zhtw-v4.md` を使って段階的中国語化 SSML を生成し Issue にコメント返信する
- 📋 `prompts/zhtw-v4.md` — Web 版 Claude セッションで 4 ループ検証した台湾華語学習教材 SSML 生成プロンプト v4 を配置。N1〜N2 学習者向け・関西弁トーン・成語解説・5項目セルフレビュー指示込み

- 📋 README に「学習教材を作る(Issue 起点ワークフロー)」セクション追加

## [0.2.0] - 2026-06-14

### 追加
- 🇯🇵🇹🇼 **日本語の声と中国語の声をそれぞれ独立して選択可能**に（プルダウン 2 つに分離）
- 🐢🐇 **日本語の速度と中国語の速度を独立スライダー化**（各 0.5〜1.5）
- 🥟 中国語の声のデフォルトを **台湾華語 (`cmn-TW-Wavenet-A`)** に変更（大陸普通話も選択肢として残置）

### 変更（内部）
- 🪄 **`<lang>` → `<voice>+<prosody>` 自動変換ロジック** (`buildSsml()`) を導入。Google TTS の `<lang>` タグが "best effort" で日本語声に中国語を読ませると無視される問題を、アプリ側で吸収。ユーザーは今まで通り Claude が出す `<lang>` 入り SSML を貼るだけで OK
- 📊 速度は入れ子 `<prosody>` で外側=ja・内側=zh/ja の相対値とし、ユーザー指定値がそのまま絶対速度として実音に反映されるよう調整

### ドキュメント
- README に「仕組み（なぜ `<lang>` で動くのか）」の節を追加
- SPEC §5 注釈を実機検証結論で更新、§6 サンプルを繁体字に統一、§7 UI 要素を 4 項目構成に更新
